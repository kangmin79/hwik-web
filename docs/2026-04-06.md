## ✅ 오후 — 태그 시스템 전면 개편 + 위치 좌표 매칭

### 태그 시스템 설계 방향 확정
- 목표: 휙에 저장되는 모든 것(매물/손님/메모)을 태그화
- 태그로 못 잡으면 Claude API(Haiku) 폴백
- 새 태그 범주는 tags 마스터 테이블에 행 추가 방식 (테이블 생성 X)

### 태그 시스템 버그 수정 4건
1. `tags.ts`: `tags.push('서울')` 하드코딩 제거 → 인천/경기/부산 등 광역시도 자동 감지
2. `batch-generate-tags`: `.or('tags.is.null,tags.eq.[]')` 쿼리 → JS 필터로 교체 (태그 2개 이하 포함)
3. `hub-new`: `_featColors`를 TAG_STYLES 앞으로 이동 + 자동 병합 (역세권/남향 등 60+ 태그 색상 정상 표시)
4. `hub-new`: `registerClient()` 후 `_updateClientTags()` 호출 추가 (손님 등록 시 태그 즉시 보완)

### 전국 구/동 태그 완성
- 부산 16개 구/군, 대구 8개, 대전 5개, 광주 5개, 울산 5개 구 추가
- 경기 주요 시 내 구 추가 (분당구/기흥구/수지구/해운대구 등)
- 도시명 없이 유니크 구명만 있어도 도시 자동 역추적 (해운대구→부산)
- 동(洞) 복수 감지 지원

### locate-card Edge Function (신규)

- 우선순위: 역 이름(stations 테이블) → 학교(schools 테이블) → 카카오 API → DISTRICT_COORDS fallback
- 역 이름 태그, 학교 태그 자동 추가
- 매물/손님/수정 저장 후 자동 호출

### auto-match 거리 점수 추가
- 기존: 벡터 유사도 + 카테고리 보너스
- 추가: 거리 점수 (0.5km→+0.25, 1km→+0.20, 2km→+0.12, 5km→+0.05)
- 손님 select 쿼리에 lat, lng 추가
- 기존 버그 수정: `filteredClients/clients` 변수명 오류 → `filteredTargets/targets`

### geo.ts 전국 좌표 확장

- 추가: 인천 7개, 경기 31개 시+구, 부산 11개, 대구 4개, 대전 3개, 광주 3개, 울산 3개

### 실행 순서 보장

- 좌표 저장 완료 후 매칭 실행 → 첫 등록부터 거리 점수 반영

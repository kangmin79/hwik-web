## 배경
GSC 데이터 분석 결과 (SEO 시작 후 약 3주, 한국):
- 23 클릭 / 2,954 노출 / CTR 0.78% / 평균 순위 10.21

- **표본이 3주치라 통계적 신뢰도 낮음** — 패턴 분류는 참고 수준

→ 노출/순위를 올려야 함. 두 가지 방향:
- **A. 관련 검색어 섹션 (내부 링크 강화)** — 토픽 클러스터 신호
- **B. 콘텐츠 깊이 확장 (FAQ + 서술형)** — 본문 풍부도 신호
- 백링크는 외부 작업 — 보류

## 작업 내용

### Phase A — 관련 검색어 섹션 (build_danji_pages.py)

**상위생활권 화이트리스트 (스샷 + 자동완성 검증):**
- 분당 / 일산 / 판교 / 광교 / 수지 / 동탄 / 미사 / 배곧 / 송도 / 마곡

**가중치 랜덤 (3개 룰):**
- 전세가율 ≥ 75% → 전세 60%, 매매·㎡당 각 20%
- 1년 가격 변동 ≥ 10% → 매매 60%, 전세·㎡당 각 20%
- 기본 → 균등 (34/33/33)

**결정적 시드:** 단지 ID hash → md5 → int → 빌드 재현성 보장

**페이지당 4개 앵커:**
1. 동 (있을 때) — prefix 화이트리스트 / "{동} 아파트 시세" / "{동} 아파트" 3종 변형
2. 구 (있을 때) — "{구} 아파트 시세" / "{구} 집값" 2종 변형
3. 랭킹 primary (가중치 선택)
4. 랭킹 secondary (primary와 다른 종류)

**HTML 구조:** `<nav aria-label="관련 검색어">` + `<h2>` + `<ul>`
**Schema:** ItemList JSON-LD 추가 (HTML 앵커와 1:1 일치)

### Phase B — FAQ + 서술형 (build_danji_pages.py)

**신규 FAQ 3개 (HTML + JSON-LD 동시):**
- "{단지} 최근 5년 최저가는?" — price_history 매매만 순회 (키 "_" 포함은 전세/월세 제외)
- "{단지} 주력 평형은?" — 카테고리별 거래 빈도 1위, 30% 이상일 때만
- "{단지} 주차 대수는?" — 세대당 0.3대 이상일 때만

**신규 서술형 2문장:**
- "최근 5년간 최저 ~에서 최고 ~까지 N억 범위에서 거래되었습니다"
- "거래가 가장 많은 평형은 전용 N㎡이며, 최근 거래 중 약 X%를 차지합니다"

**보일러플레이트 회피:** 데이터 없는 단지는 항목 자동 생략 (Clean Index 점수 보호)

## 발견된 데이터 이슈 (수정함)

**price_history 키 혼재:** "85"(매매), "85_jeonse"(전세), "85_wolse"(월세)가 같은 dict에 들어있음. 처음 5년 최저가 계산 시 월세 보증금이 최저가로 잡혀 "4,200만 (전용 85_wolse㎡)" 노출. 매매만 (`"_" not in str(_cat_k)`) 필터링으로 해결.

**기존 코드 문제 (수정 안 함):** `total_recent_trades` 카운트(line 460)도 매매/전세/월세 혼재 — 사실상 "최근 1년간 매매 거래 N건" 표현이 정확하지 않을 수 있음. 별도 이슈, 추후 데이터 정제 시 고민.

## 수정한 진짜 버그 4건

### 월세/보증금 필터 — DB 저장값 무시
파일: supabase/functions/auto-match/index.ts
- 문제: `filterMatch`가 손님 텍스트 재파싱에 의존. AI가 파싱한 `cp.price`에 "월 100 이하" 같은 패턴이 없으면 `parsePriceCondition`이 null 반환 → 필터 항상 통과
- 수정: 월세일 때 `client.monthly_rent`/`client.deposit` DB 값을 우선 사용. 매매·전세도 maxPrice null이면 `client.price_number` fallback

### auto-match filterMatch가 rawText 누락 (드리프트)
- 문제: match-properties는 `cp.rawText` 포함, auto-match는 빠뜨림 → "3억5천 이상 5억5천 이하" 같은 범위 조건이 파싱 실패
- 수정: `allText`에 `private_note.rawText` + `cp.rawText` + `cp.complex` + `cp.area` 추가

### parsePriceCondition이 "N억 M천 이상" 패턴 누락
파일: supabase/functions/_shared/price.ts
- 문제: minMatch 정규식이 `/(\d+\.?\d*)\s*억\s*(?:이상|...)/`로 "3억5천 이상" 같은 복합 숫자 못 잡음
- 수정: 뒤에 `(\d+)?\s*천?` 추가해서 "3억5천 이상" 지원

### auto-match 손님→매물 방향 wanted_* 필드 select 누락
- Step 1 drift 차단 작업으로 자동 해결 (card-fields.ts 도입)

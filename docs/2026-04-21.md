## 수정한 프로덕션 버그 7건

### 1. 자기 매물 trade_status 무시 (auto-match)
계약중/완료 매물이 auto-match 돌면 새 손님과 매칭 → 알림 오작동. 방향 시작 시 `trade_status !== '계약가능'` 조기 반환.

### 2. 자기 손님 client_status 무시 (auto-match)
연락두절/계약완료 손님에게 매물 알림 감. `client_status` 체크 + target 필터 `.in('client_status', [탐색중,급해요,협의중])`. `MATCH_CLIENT_EXTRA`에 `client_status` 추가.

### 3. 복수 거래유형("매매나 전세") 미지원 (auto-match)
`wanted_trade_type` 단일 필드만 검사 → `wanted_conditions[].trade_type` 배열 수집해서 OR 판정.

### 4. 카테고리 필터 과잉 (parse-property + auto-match)
손님 "20평 월세"만 적어도 `parsedResult.category="apartment"` 기본값이 `wanted_categories` 주입 → 원룸 매물 탈락. 주입 제거 + `wanted_categories` 빈 배열이면 전체 허용.

### 5. 전세/매매 구분 버그 (parse-property L698)
"거래유형 키워드 없고 억 단위" → 무조건 "전세"로 설정했음. "헬리오시티 25억 이하" 같은 단문이 전세로 오분류. 기본값을 "매매"로 수정.

### 6. Anthropic 500 Overloaded 21% 실패율 (parse-property)
매물 등록 5개 중 1개 실패 수준. `callAnthropicWithRetry` (529/503/500 4회 exponential backoff) + 프롬프트 캐싱(`cache_control: ephemeral`) — 비용 90%↓ + Overloaded 빈도 감소.

### 7. 인접구 false-positive (auto-match 지역 필터)
좌표 있을 때 구 이름 무시하고 5km 반경 매칭 → 송파↔강동, 마포↔용산 등. 구 이름 양쪽 다 있으면 엄격 비교, 매물 구 미상일 때만 좌표 폴백. `GU_RE`에 경기/인천 도시 추가.

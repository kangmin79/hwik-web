## 발견한 클로킹 메커니즘 (재발 방지 핵심)
오전 SEO 작업의 구조:
```html
<div class="wrap" id="app">
  <div class="loading"></div>
  <div id="fallback-content" style="display:none;">  ← inline display:none
    ...FAQ 11 + 서술형 + 관련 검색어...
  </div>
</div>
```
+ style.css 라인 145: `#fallback-content { display:none; }` ← CSS도 이중 숨김
+ danji/app.js 라인 574: `$('#app').innerHTML = ...` ← #app 통째로 덮어씀

**결과:**
- bot이 보는 HTML: FAQPage JSON-LD 11개 + fallback HTML
- 사용자가 보는 화면: app.js가 그린 별개 콘텐츠 (FAQ 텍스트 다름)
- fallback은 페이지 로드 후 DOM에서 완전히 사라짐

**Google 정책 위반:**
- "Don't mark up content that is not visible to readers of the page"
- FAQPage 특히 엄격: 모든 Q&A가 페이지에 visible 해야 함
- 봇/사용자에게 다른 콘텐츠 = 클로킹

## 수정 시도가 누적 실패한 이유
1. inline display:none만 제거 → CSS 규칙으로 여전히 숨김 (놓침)
2. CSS 규칙 제거 후 노출 → SPA가 그린 동적 콘텐츠와 중복
3. 중복 제거하다 보니 페이지 시각이 누적 변경 → 사용자 어색함
4. SPA의 .seo-text + seoParts (별개 SEO 텍스트)도 fallback과 중복
5. fallback의 "주변 단지"도 SPA와 중복
6. 데이터 안내 박스 위치 어색 → fallback 끝으로 이동 시도

각 수정마다 새 중복/위치 문제 발견. 사용자가 누적 변경에 지쳐 롤백 결정.

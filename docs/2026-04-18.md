## 발견한 버그 3가지

### 1. `numOfRows` 파라미터 무시 (가장 치명)
전유공용면적 API가 `numOfRows=1000` 요청해도 **페이지당 고정 100건만 반환**.  
기존 코드 `range(1, 21)` 상한 → 실제 **최대 2,000건만 수집**.  
헬리오시티(totalCount 106,959) → 2,000건만 가져와서 **98% 누락** 상태.

### 2. 조기 종료 로직이 희귀 타입 버림
`no_new_type_pages >= 3` 조건이 3호 미만 희귀 타입(예: 사이드 펜트하우스)을 놓침.  
동일 단지 두 번 돌리면 결과가 다를 정도로 불안정.

### 3. K-apt `kaptAddr`가 건축물대장 실제 번지와 불일치
헬리오시티: kaptAddr "가락동 479" → API totalCount=0.  
실제 지번은 **가락동 913** (doroJuso 역변환으로 확인).  
대단지·재건축 단지 공통 이슈.

## 오늘 작업 요약

### 1. 검색 개선 (이어서)

#### 가격 단독 입력 → "정도"로 처리
- "4억" → 기존: 4억 이하(0~4억) → 수정: 4억 정도(3.4~4.6억)
- 중개사가 "4억"이라고 치면 4억 근처를 원하는 것

#### 가격 키워드 대폭 확장
- 이하 계열: +밑으로, 내로, 안넘는, 못넘는
- 이상 계열: +위로, 넘게
- 정도 계열: +선, 대, 안팎, 전후, 언저리
- 오타: +이해→이하, 안됨→이하

#### 검색어에 위치 없으면 → 사무실 주소 자동 적용
- profiles.address에서 구 이름 추출
- "아파트 전세 5억" → 사무실이 마포구면 "마포구 아파트 전세 5억"으로 처리

#### 검색어 단어별 AND/OR 매칭
- "역삼 래미안 전세" → search_text에 '역삼' AND '래미안' 포함
- AND로 0건이면 OR로 재시도 ("강남 서초" → 강남 OR 서초)
- 기존 location 파싱 대신 텍스트 직접 매칭

#### 위치 필터 근본 수정
- bounding box(좌표 사각형) → property.location 텍스트 ILIKE 필터
- 구 경계가 불규칙해서 bounding box는 근본적으로 부정확
- 텍스트 필터가 100% 정확

#### 위치 가중치 항상 4배
- 위치를 검색했으면 위치가 가장 중요 (10점 → 40점)
- 가격 조건 유무와 관계없이

### 10. 401 에러 디버깅 과정

#### 원인
- 보안 수정에서 `SUPABASE_KEY → session.access_token` 교체
- `window._accessToken`이 undefined → `'Bearer undefined'` → Supabase 게이트웨이 401
- 서버에 추가한 토큰 검증(`getUser`) → anon key로 호출 시 에러

#### 해결
- my_cards.html: match-properties 호출에 `SUPABASE_KEY` 복원 (유효한 JWT)
- Edge Function: 불필요한 토큰 검증 로직 완전 제거
- `agent_id`로만 권한 검증 (서비스 키로 DB 접근하므로 충분)

#### 교훈
- Supabase Edge Function은 `Authorization` + `apikey` 두 헤더 필수
- `Authorization`에 유효한 JWT 아니면 게이트웨이가 401 반환
- 보안 강화하려다 기능 깨뜨린 케이스 — 작동하는 것부터 확인 후 강화

### 11. CORS 제한 적용

### 14. 월세 보증금/월세금 별도 매칭 (핵심!)

#### 문제
```
손님: "보증금 500 월세 80 이내"
  → max_price = 80 (월세금)
매물: "보증금500/월80"
  → price_number = 500 (보증금)
비교: 500 > 88 → "가격 초과" → 매칭 0건!
```

#### 해결
- 손님에서 `wantedDeposit`/`wantedMonthly` 각각 추출
- 매물 가격 문자열에서 보증금/월세금 실시간 파싱
- 보증금 비교(25점) + 월세금 비교(25점) = 최대 50점
- 10% 이내 만점, 30% 이내 15점, 초과 -10점

#### 결과
- 서대문 사무실 월세: 0건 → **5건** 매칭 성공

## B. agent.html 중개사 홈 변화 (14 커밋)

### 0. 발견
어제(`855eba2bb33`) 이미 중개사 홈 기본 뼈대가 있었음. URL: `/agent.html?id={agent_id}[&kapt_code=...&type=...]`

### 1. 인벤토리 전체 노출 방지 (`9ba59bee53e`)
- 문제: `?id=` 단독 방문 시 100건 전 매물 노출 → 경쟁 중개사 정찰·손님 피로
- 해결: URL 파라미터별 필터
  - `?id=&kapt_code=&type=` → 해당 단지+거래유형만 (기존)
  - `?id=&cards=` → 공유된 카드만 (기존)
  - `?id=&lat=&lng=` → 좌표 주변 5건 (레거시)
  - `?id=` 단독 → **거래유형별 대표 1건 + 카톡 상담 CTA**

### 2. 헤더 신뢰형 풀 리디자인 (`cb3d0bd93d2` → `6cc82ee686d`)
- 다크 네이비 → 화이트 카드 + 프로필 사진 강조
- 전화 문의 = 아웃라인(보조), 카톡 = `#FEE500` 브랜드 컬러(주 CTA)
- 컴팩트화: 프로필 72→46px, 패딩 축소, 이름 16px

### 3. 정보 구조 정돈 (`b1702fc8d9d` → `412e0033b08`)
- 주소를 프로필 정보 하단으로 이동 (이전: CTA 아래 허공)
- 지역을 태그로: `강동욱 [공인중개사] [중랑구 전문]` 한 줄에

### 4. 전문 지역 최대 3개 (`ca2cfbb6459`)

## C. hub-new 프로필 편집 버그 3개 수정

### 1. 저장 후 모달 안 닫힘 (`8be7853d2d4`)
- `_saveSettings`에 `editModal.classList.remove('show')` 누락
- 동시에 upsert 에러 응답 체크 추가 (숨겨진 실패 노출)

### 2. NOT NULL 제약 위반 (`260a18256e8`)
- Supabase upsert = INSERT...ON CONFLICT. NOT NULL은 INSERT 단계에서 먼저 검사.
- `_saveSettings`의 upsert에 `phone` 누락 → "저장 실패: phone violates not-null"
- 해결: `phone:Auth.profile?.phone||''` 추가

### 3. 취소/저장 버튼 크기 비대칭 (`79d06ece5b9`)
- 인라인 `style="width:auto;padding:10px 24px"`가 `.edit-buttons`의 `flex:1` CSS 덮어씀
- 두 모달의 인라인 스타일 제거 → 동일 크기

# 2026-03-28 단지 페이지 전체 작업 기록

## 1. 데이터 파이프라인 구축

### 실거래가 수집 (sync_trades.py)
- 서울 3개 구(강남/송파/강동) 3년치 125만건 수집
- 전세/월세 분리: monthlyRent > 0 → 월세, 아니면 전세
- 월세 price_history에 monthly(월세금) 필드 추가
- --aggregate-only 옵션 추가 (수집 없이 집계만)
- categories: 건축물대장 전체 → 실거래 있는 평형만
- danji_pages ID: kapt_code 뒤 4자리 suffix로 고유성 보장
- 주변 단지: 같은 property_type + ±10㎡ 평형별 가격 저장
- sitemap.xml 자동 생성
- GitHub Actions: 매일 새벽 1시(KST) 자동 동기화

### 공급면적 수집 (sync_pyeongs.py)
- 건축물대장 전유공용면적 API (getBrExposPubuseAreaInfo)
- 주거공용만 필터: 복도, 계단, 승강기, 홀, 현관, 통로, 벽체
- 기타공용 제외: 주차장, 대피소, 관리사무소, 기계실 등
- 결과: 전용:공급 비율 1.3배 (리치고와 일치)
- 이전 방식(전체 공용 합산)은 2배로 잘못됨 → 수정 완료

### 엑셀 임포트 (import_apt_excel.py)
- K-apt 기본정보: 주차/최고층/난방/시공사
- K-apt 관리비: 공용관리비만 (개별사용료 제외) → 결국 표시 제거
- 3~6개월마다 새 엑셀로 재실행

## 3. danji.html 기능

### 지표 (6개 박스)
- 1행: 최근 실거래, 3년 내 최고가, 전세가율
- 2행: 평당가, 최고 대비 %, 최근 6개월 거래량
- 월세 탭: 지표 전체 숨김

### 면적 표시
- 전용면적 기본 표시 (㎡만, 평 없음)
- 공급면적 토글: [전용] [공급] 버튼 (공급 데이터 있을 때만)
- 공급면적: 건축물대장 주거공용만 필터 (정확)
- 선택된 면적 버튼 항상 맨 왼쪽
- 기본 선택: 84㎡(국민평수)에 가장 가까운 것

### 차트
- scatter (점 1개 = 거래 1건)
- 점 개수에 따라 크기/투명도 자동 조절
- 거래량 바 차트 (위)
- 월세 탭: 차트 숨김 (보증금/월세 두 축 표현 불가)
- 터치: 점 근처 아무 곳 터치해도 툴팁
- 툴팁: 날짜, 면적, 가격 + 층수

### 거래 목록
- 현재 선택 평형 + 탭 기준 필터
- 월세: "보증금 / 월세금" 형태
- 5건 표시

### 주변 단지
- 현재 선택 평형 ±10㎡ 기준 비교
- 비슷한 평형 없으면 제외
- 평당가 순 정렬 (비싼 순)
- 평당가 표시
- 비쌈/저렴 판단 제거 (위치 등 차이)

### FAQ
- 데이터 있는 항목만 표시
- 실거래가, 평당가, 전세가율, 지하철, 3년 내 최고가, 주차, 난방, 시공사

### UI
- 좌측 컬러바 (노랑/빨강/파랑/초록)
- 터치 피드백 (scale 0.97)
- localStorage 캐싱 (stale-while-revalidate)
- 지하철(파랑) / 학교(초록) 태그 구분
